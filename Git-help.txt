$ git init - Эта команда создаёт в текущем каталоге 
			новый подкаталог с именем .git содержащий все необходимые 
			файлы репозитория — основу Git-репозитория.
			
$ git clone https://github.com/FujiF/Test_repo.git - Клонирование репозитория
из сети

$ git add - индексирует файлы в рабочего каталога

$ git commit -a -m(message 'hello')- заставляет Git автоматически индексировать
каждый уже отслеживаемый на момент коммита файл, позволяя вам обойтись без git add.

$ git rm - удаляет файл из вашего рабочего каталога

$ git rm --cached readme.txt -  вы можете захотеть оставить файл на винчестере,
 и убрать его из-под бдительного ока Git'а
 
$ git mv file_from file_to - переименовать файл

$ git log (--stat - более подробно)- выводит список коммитов созданных в данном 
									 репозитории
									 
$ git log --pretty=format:"%h - %an, %ar : %s"
ca82a6d - Scott Chacon, 11 months ago : changed the version number
Параметр    Описание выводимых данных
%H  Хеш коммита %h  Сокращённый хеш коммита
%T  Хеш дерева %t  Сокращённый хеш дерева
%P  Хеши родительских коммитов %p  Сокращённые хеши родительских коммитов
%an Имя автора %ae Электронная почта автора
%ad Дата автора (формат соответствует параметру --date= )
%ar Дата автора, относительная (пр. "2 мес. назад")
%cn Имя коммитера %ce Электронная почта коммитера
%cd Дата коммитера %cr Дата коммитера, относительная
%s  Комментарий

$ gitk - Визуализация истории

$ git commit --amend -  когда вы делаете коммит слишком рано, забыв добавить 
какие-то файлы, или напутали с комментарием к коммиту. Если вам хотелось бы 
сделать этот коммит ещё раз,(Вы добавляете в индекс изменения, которые хотите, 
редактируя файл и выполняя для него git add или выполняя git rm для отслеживаемого 
файла, и затем git commit --amend возьмёт текущий индекс и сделает его снимком 
состояния нового коммита.)

$ git reset HEAD <файл> - Отмена индексации файла.

$ git checkout -- <файл> - Отмена изменений файла.

$ git remote (-v) - Отображение удалённых репозиториев. Чтобы посмотреть, 
какому URL соответствует сокращённое имя в Git, можно указать команде опцию -v.

$ git remote add Helper(Repository name) https://github.com/FujiF/New.git - 
										Добавление удалённых репозиториев

$ git fetch name(имя удал. сервера) - извлекает оттуда все данные, которых у вас ещё нет, 
и обновляет ваше локальное хранилище данных

$ git push [удал. сервер](Helper) [ветка](master) - отправить в главный репозиторий

$ git remote show [удал. сервер] - Инспекция удалённого репозитория

$ git remote rename oldname newname переименование удалённых репозиториев

$ git remote rm [удал. сервер](Helper) - Удаление удалённого репозитория

$ git tag -a v1.4 -m 'my version 1.4' - Аннотированные метки

$ git show v1.4 - посмотреть данные метки вместе с коммитом

$ git config --global alias.ci commit - Псевдонимы в Git

$ git branch branch_name - создадим новую ветку branch_name

$ git checkout branch_name - перейдём на новую ветку branch_name
$ git checkout -b branch_name -  создать ветку и сразу же перейти на неё

$ git branch -d branch_name - удалить ветку

$ git checkout master 
$ git merge branch_name - Вы можете запустить тесты, убедиться, что решение 
работает, и слить (merge) изменения назад в ветку master, чтобы включить их в продукт.

$ git mergetool -  графический инструмент и покажет конфликтные ситуации

$ git branch -v - посмотреть последний коммит на каждой из веток
$ git branch --merged - посмотреть те ветки, которые вы уже слили с текущей
$ git branch --no-merged - увидеть все ветки, содержащие наработки, 
которые вы пока ещё не слили в текущую ветку

$ git push origin serverfix:awesomebranch. Так ваша локальная ветка serverfix 
отправится в ветку awesomebranch удалённого проекта.

$ git push [удал. сервер] :[ветка] - удалить ветку на сервере

$ git add -i - Git перейдёт в режим интерактивной оболочки(Интерактивное индексирование)
			-Если набрать 2 или u в приглашении What now>, сценарий спросит, 
			какие файлы вы хотите добавить в индекс.
			-Если в этот момент вы хотите удалить файл из индекса, 
			используйте опцию 3 или r (revert).
			- Чтобы посмотреть дельту для проиндексированных изменений, 
			используйте команду 6 или d.
			- Индексирование по частям - глава книги 6.2
			
$ git stash - Прятанье
$ git stash list - Чтобы посмотреть, что у вас есть припрятанного	
$ git stash apply - Вы можете снова применить ту, которую только что спрятали
git stash apply stash@{Nomber} -  Если вы хотите применить одну из старых заначек, 
можете сделать это, указав её имя так
$ git stash apply --index - Изменения в файлах были восстановлены, но файлы в индексе — нет. 
Чтобы добиться такого, необходимо выполнить команду.тогда команда попытается применить 
изменения в индексе	
$ git stash drop stash@{0} - Чтобы удалить спрятанное
$ git stash pop - чтобы применить спрятанные изменения и сразу же удалить их из стека.
$ git stash show -p stash@{0} | git apply -R - Откат применения спрятанных изменений
